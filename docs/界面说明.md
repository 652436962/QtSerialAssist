## 三、左侧面板 (leftPanel)

宽度 200px（最小），可通过 mainSplitter 拖拽调整。包含：QTabWidget、工具栏。顶部为 QTabWidget，底部为工具栏面板，中间有垂直弹性空间。

### 3.1 QTabWidget - 串口设置页 (serialTab)

网格布局（6 行 × 2 列）：

| 行 | 标签 (QLabel) | 控件 | 对象名 | 可选值 |
|----|--------------|------|--------|--------|
| 1 | 串口 | QComboBox | `serialPortCombo` | 自动检测可用串口，2 秒自动刷新 |
| 2 | 波特率 | QComboBox | `baudRateCombo` | 9600, 19200, 38400, 57600, 115200, 460800, 921600 |
| 3 | 数据位 | QComboBox | `dataBitsCombo` | 5, 6, 7, 8 |
| 4 | 停止位 | QComboBox | `stopBitsCombo` | 1, 1.5, 2 |
| 5 | 校验位 | QComboBox | `parityCombo` | None, Odd, Even, Mark, Space |
| 6 | 流控 | QComboBox | `flowControlCombo` | None, Hardware, Software |

> **注**: 串口端口列表通过 `QSerialPortInfo::availablePorts()` 获取，显示格式为 "端口名 - 描述"。未安装 Qt SerialPort 库时此页自动禁用。

### 3.2 QTabWidget - 网络设置页 (networkTab)

网格布局（3 组控件）：

| 行 | 标签 | 控件 | 对象名 | 说明 |
|----|------|------|--------|------|
| 1 | 协议 | QComboBox | `protocolCombo` | TCP 服务器 / TCP 客户端 / UDP |
| 2 | IP 地址 | QLineEdit | `ipLineEdit` | 默认 192.168.1.100 |
| 3 | 端口 | QSpinBox | `portSpinBox` | 默认 8080 |

> **连接逻辑**:
> - TCP 服务器: 监听指定端口，接受客户端连接
> - TCP 客户端: 连接指定 IP:Port，超时 3 秒
> - UDP: 绑定本地端口，通过 IP:Port 发送数据报

### 3.3 打开连接按钮

| 控件 | 对象名 | 位置 | 说明 |
|------|--------|------|------|
| 串口连接 | `connectBtn` | serialTab 底部 | 点击连接/断开串口 |
| 网络连接 | `connectNetBtn` | networkTab 底部 | 点击连接/断开网络 |

> 两个按钮均连接到同一个 `on_connectBtn_clicked` 槽函数，根据当前 Tab 页判断连接类型。

## 四、右侧面板 (rightPanel)

右侧面板通过 `rightSplitter`（垂直分割器）分为上下两部分：上半部分为接收数据区域，下半部分为 `modbusSplitter`（水平分割器），包含 Modbus 面板、快捷指令面板和发送数据区域。

### 4.1 接收数据区域 (receiveWidget)

| 子控件 | 对象名 | 类型 | 说明 |
|--------|--------|------|------|
| 标题 | `receiveLabel` | QLabel | "接收数据"，高度 30px |
| 显示区 | `receiveTextEdit` | QTextEdit | 只读；深色背景 + 等宽字体 |
| 统计 | `receiveBytesLabel` | QLabel | 显示 "接收: X 字节" |
| 统计 | `frameCountLabel` | QLabel | 显示 "帧数: X" |
| 清空 | `clearReceiveBtn` | QPushButton | 位于标题行右侧，清空接收显示区并重置统计 |

**显示格式**:
```
[时:分:秒.毫秒] HEX_DATA
```
- 发送的数据显示为绿色 (`#4ecca3`)
- 接收的数据显示为蓝色 (`#6eb5ff`)
- 数据以十六进制大写格式显示，每字节间有空格分隔

### 4.2 Modbus 面板 (modbusPanelWidget)

默认隐藏，通过工具栏 "Modbus" 按钮切换显示。位于 modbusSplitter 最左侧，最小宽度 250px。

| 子控件 | 对象名 | 类型 | 说明 |
|--------|--------|------|------|
| 标题 | `modbusPanelLabel` | QLabel | "Modbus" |
| 格式选择 | `modbusFormatCombo` | QComboBox | RTU / ASCII / TCP |
| 从站地址 | `modbusSlaveSpinBox` | QSpinBox | 范围 1~247，默认 1 |
| 功能码 | `modbusFuncCombo` | QComboBox | 01 读线圈 / 03 读保持寄存器 / 05 写单个线圈 / 06 写单个寄存器 / 10 批量写寄存器 |
| 寄存器地址 | `modbusAddrSpinBox` | QSpinBox | 范围 0~65535 |
| 读写数量 | `modbusCountSpinBox` | QSpinBox | 范围 1~125，默认 1 |
| 发送按钮 | `modbusSendBtn` | QPushButton | 发送 Modbus 请求 |
| 更新按钮 | `modbusUpdateBtn` | QPushButton | 生成请求数据包并显示 |
| 请求数据显示 | `modbusResponseEdit` | QTextEdit | 显示生成的请求数据包（占位符 "请求数据..."） |

**数据包格式**:
- RTU 模式: 从站地址 + 功能码 + 数据 + CRC16 校验
- ASCII 模式: `:` 开头 + 十六进制帧 + LRC 校验 + `\r\n` 结尾
- TCP 模式: MBAP 头（事务ID + 协议ID + 长度）+ PDU

### 4.3 快捷指令面板 (quickCmdPanelWidget)

默认隐藏，通过工具栏 "快捷指令" 按钮切换显示。位于 modbusSplitter 中间，最小宽度 300px。指令配置以 JSON 文件存储在 `commands/` 目录下。

| 子控件 | 对象名 | 类型 | 说明 |
|--------|--------|------|------|

| 添加分组 | `quickCmdAddGroupBtn` | QPushButton | "+" 按钮，添加新分组 |
| 删除分组 | `quickCmdDelGroupBtn` | QPushButton | "-" 按钮，删除当前分组 |
| 延时设置 | `quickCmdDelaySpinBox` | QSpinBox | 范围 0~60000ms，默认 500ms |
| 指令表格 | `quickCmdTable` | QTableWidget | 4 列：勾选框 / 名称 / 数据 / 加载按钮 |
| 添加按钮 | `quickCmdAddBtn` | QPushButton | 添加新指令到当前分组 |
| 删除按钮 | `quickCmdDelBtn` | QPushButton | 删除勾选的指令 |
| 全选按钮 | `quickCmdSelectAllBtn` | QPushButton | 全选所有指令 |

- 点击 "加载" 按钮将对应指令数据填入发送区

### 4.4 发送数据区域 (sendWidget)

位于 modbusSplitter 最右侧，占据剩余空间。发送设置（模式/换行/编码）已整合到标题栏中。

| 子控件 | 对象名 | 类型 | 说明 |
|--------|--------|------|------|

| 编码 | `encodingCombo` | QComboBox | UTF-8 / GBK |

### 5.1 按钮状态

| 模式 | 按钮文字 | 按钮样式 |
|------|----------|----------|
| 普通模式 | "终端模式" | 默认主题样式 |
| 终端模式 | "普通模式" | 红色背景 (`#e94560`)，白色文字 |

### 5.2 键盘交互

| 按键 | 行为 |
|------|------|
| Enter / Return | 根据换行设置（`\r\n` / `\n` / `\r`）发送换行符到连接 |
| Backspace | 仅在本地删除字符，不发送到远端 |
| 可打印字符 | 以绿色 (`#4ecca3`) 显示在接收区，同时发送到当前连接 |
| 方向键 | 正常导航光标位置 |

### 5.3 终端模式下的发送

用户输入的字符通过 `sendTerminalData()` 函数路由到当前活动连接：
- **串口模式**: 通过 `QSerialPort::write()` 发送
- **TCP 模式**: 通过 `QTcpSocket::write()` 发送
- **UDP 模式**: 通过 `QUdpSocket::writeDatagram()` 发送

### 5.4 实现要点

- `QSplitter::setCollapsible(1, true)` 允许底部面板折叠，`hide()` 后自动释放空间
- `eventFilter()` 捕获 `QEvent::KeyPress`，区分 Enter/Backspace/可打印字符
- 进入终端模式时 `receiveTextEdit` 设为可编辑并获取焦点
- 退出终端模式时恢复只读状态并移除事件过滤器

## 八、核心交互流程

### 连接流程
```
用户选择 Tab 页（串口/网络）
  → 配置参数（端口、波特率 / IP、端口）
  → 点击 "打开连接"
  → 成功后按钮变为 "关闭连接"，状态栏显示详情
  → 再次点击断开连接
```

### 发送流程
```
用户在发送区输入十六进制数据
  → 点击 "发送" / "发送新行"
  → parseHexInput() 解析为字节数组
  → 根据当前连接类型（串口/TCP/UDP）写入数据
  → 成功后在接收区回显（绿色时间戳）
  → 更新发送字节计数
```

### 接收流程
```
ReadyRead 信号触发
  → readAll() 读取数据
  → 更新接收字节计数和帧数
  → 转换为十六进制大写格式
  → 追加到接收显示区（蓝色时间戳）
```

### 定时发送流程
```
勾选 "定时发送" CheckBox
  → QTimer 按设定间隔触发
  → 自动调用 on_sendBtn_clicked()
```
