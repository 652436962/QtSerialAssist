## 三、左侧面板 (leftPanel)

宽度 200px（最小），可通过 mainSplitter 拖拽调整。包含：QTabWidget、工具栏。顶部为 QTabWidget，底部为工具栏面板，中间有垂直弹性空间。

### 3.1 QTabWidget - 串口设置页 (serialTab)

网格布局（6 行 × 2 列）：

| 行 | 标签 (QLabel) | 控件 | 对象名 | 可选值 |
|----|--------------|------|--------|--------|
| 1 | 串口 | QComboBox | `serialPortCombo` | 自动检测可用串口，2 秒自动刷新 |
| 2 | 波特率 | QComboBox | `baudRateCombo` | 9600, 19200, 38400, 57600, 115200, 460800, 921600 |
| 3 | 数据位 | QComboBox | `dataBitsCombo` | 5, 6, 7, 8 |
| 4 | 停止位 | QComboBox | `stopBitsCombo` | 1, 1.5, 2 |
| 5 | 校验位 | QComboBox | `parityCombo` | None, Odd, Even, Mark, Space |
| 6 | 流控 | QComboBox | `flowControlCombo` | None, Hardware, Software |

> **注**: 串口端口列表通过 `QSerialPortInfo::availablePorts()` 获取，显示格式为 "端口名 - 描述"。未安装 Qt SerialPort 库时此页自动禁用。

### 3.2 QTabWidget - 网络设置页 (networkTab)

网格布局（3 组控件）：

| 行 | 标签 | 控件 | 对象名 | 说明 |
|----|------|------|--------|------|
| 1 | 协议 | QComboBox | `protocolCombo` | TCP 服务器 / TCP 客户端 / UDP |
| 2 | IP 地址 | QLineEdit | `ipLineEdit` | 默认 192.168.1.100 |
| 3 | 端口 | QSpinBox | `portSpinBox` | 默认 8080 |

> **连接逻辑**:
> - TCP 服务器: 监听指定端口，接受客户端连接
> - TCP 客户端: 连接指定 IP:Port，超时 3 秒
> - UDP: 绑定本地端口，通过 IP:Port 发送数据报

### 3.3 打开连接按钮

| 控件 | 对象名 | 位置 | 说明 |
|------|--------|------|------|
| 串口连接 | `connectBtn` | serialTab 底部 | 点击连接/断开串口 |
| 网络连接 | `connectNetBtn` | networkTab 底部 | 点击连接/断开网络 |

> 两个按钮均连接到同一个 `on_connectBtn_clicked` 槽函数，根据当前 Tab 页判断连接类型。

## 四、右侧面板 (RightPanel.qml)

右侧面板通过垂直布局分为上下两部分：上半部分为接收数据区域（ScrollView），下半部分为发送数据区域。

### 4.1 接收数据区域 (receiveScroll)

| 子控件 | 对象名 | 类型 | 说明 |
|--------|--------|------|------|
| 标题 | — | Text | "接收数据"，高度 20px |
| 显示区 | `receiveArea` | TextArea | 只读（终端模式可编辑）；等宽字体 13px |
| 滚动容器 | `receiveScroll` | ScrollView | 包裹 receiveArea 提供垂直滚动 |

**显示格式**:
- 发送的数据以 `---> ` 前缀标识
- 接收的数据以 `<--- ` 前缀标识
- 收到新数据时自动滚动到底部（可通过设置关闭）
- 终端模式下接收区可编辑，直接输入字符发送到当前连接

**布局稳定性**:
- `receiveArea.width` 绑定到 `receiveScroll.width`，防止内容撑大布局
- 发送窗口高度硬约束 `Layout.minimumHeight/maximumHeight: 160`，接收区填满剩余空间

### 4.3 Modbus 面板 (ModbusPanel.qml)

通过点击标题栏 "Modbus" 按钮切换显示在左侧面板。

| 子控件 | 类型 | 说明 |
|--------|------|------|
| 格式选择 | `DComboBox` | RTU / ASCII / TCP |
| 从站地址 | `SpinBox` | 范围 1~247，默认 1 |
| 功能码 | `DComboBox` | 01 读线圈 ~ 10 批量写寄存器 |
| 寄存器地址 | `SpinBox` | 范围 0~65535 |
| 数量 | `SpinBox` | 范围 1~125，默认 1 |
| 生成帧 | `DButton` | 生成 Modbus 请求帧 |
| 结果 | `TextArea` | 只读，显示生成的请求帧 |

**数据包格式**:
- RTU 模式: 从站地址 + 功能码 + 数据 + CRC16 校验
- ASCII 模式: `:` 开头 + 十六进制帧 + LRC 校验 + `\r\n` 结尾
- TCP 模式: MBAP 头（事务ID + 协议ID + 长度）+ PDU

### 4.4 快捷指令面板 (QuickCmdPanel.qml)

通过点击标题栏 "快捷命令" 按钮切换显示在左侧面板。指令配置以 JSON 文件存储在 `commands/` 目录下。

| 子控件 | 类型 | 说明 |
|--------|------|------|
| 分组选择 | `DComboBox` | 下拉切换指令分组 |
| 添加/删除分组 | `DButton` | "+" 添加分组，"-" 删除当前分组 |
| 指令列表 | `ListView` | 显示名称、数据、发送按钮 |
| 添加/删除指令 | `DButton` | 底部操作栏 |
| 全选/发送全部 | `DButton` | 批量操作 |

**交互逻辑**:
- 点击 "▸" 按钮直接发送对应指令数据
- 点击 "发送全部" 依次发送所有指令
- 分组和指令自动保存到 `commands/<分组名>.json`

### 4.5 发送数据区域 (sendArea)

位于右侧面板底部，高度固定 160px。

| 子控件 | 对象名 | 类型 | 说明 |
|--------|--------|------|------|
| 标题 | — | Text | "发送数据" |
| 模式 | — | DButton | HEX / ASCII 切换 |
| 发送按钮 | — | DButton | 解析并发送数据 |
| 发送+换行 | — | DButton | 发送数据 + `\r\n` |
| 输入区 | `sendArea` | TextArea | 等宽字体，placeholderText 提示 |
| 定时发送 | — | DSwitch | 勾选后按间隔自动发送 |
| 间隔设置 | `autoSendInterval` | SpinBox | 100~60000ms，默认 1000ms |

**发送数据格式**: 支持十六进制格式输入（空格分隔），ASCII 模式下根据编码设置转换文本。Ctrl+Enter 快捷键发送。

### 5.1 按钮状态

| 模式 | 按钮文字 | 按钮样式 |
|------|----------|----------|
| 普通模式 | "终端模式" | 默认主题样式 |
| 终端模式 | "普通模式" | 红色背景 (`#e94560`)，白色文字 |

### 5.2 键盘交互

| 按键 | 行为 |
|------|------|
| Enter / Return | 根据换行设置（`\r\n` / `\n` / `\r`）发送换行符到连接 |
| Backspace | 仅在本地删除字符，不发送到远端 |
| 可打印字符 | 以绿色 (`#4ecca3`) 显示在接收区，同时发送到当前连接 |
| 方向键 | 正常导航光标位置 |

### 5.3 终端模式下的发送

用户输入的字符通过 `sendTerminalData()` 函数路由到当前活动连接：
- **串口模式**: 通过 `QSerialPort::write()` 发送
- **TCP 模式**: 通过 `QTcpSocket::write()` 发送
- **UDP 模式**: 通过 `QUdpSocket::writeDatagram()` 发送

### 5.4 实现要点

- `QSplitter::setCollapsible(1, true)` 允许底部面板折叠，`hide()` 后自动释放空间
- `eventFilter()` 捕获 `QEvent::KeyPress`，区分 Enter/Backspace/可打印字符
- 进入终端模式时 `receiveTextEdit` 设为可编辑并获取焦点
- 退出终端模式时恢复只读状态并移除事件过滤器

## 八、核心交互流程

### 连接流程
```
用户选择 Tab 页（串口/网络）
  → 配置参数（端口、波特率 / IP、端口）
  → 点击 "打开连接"
  → 成功后按钮变为 "关闭连接"，状态栏显示详情
  → 再次点击断开连接
```

### 发送流程
```
用户在发送区输入十六进制数据
  → 点击 "发送" / "发送新行"
  → parseHexInput() 解析为字节数组
  → 根据当前连接类型（串口/TCP/UDP）写入数据
  → 成功后在接收区回显（绿色时间戳）
  → 更新发送字节计数
```

### 接收流程
```
ReadyRead 信号触发
  → readAll() 读取数据
  → 更新接收字节计数和帧数
  → 转换为十六进制大写格式
  → 追加到接收显示区（蓝色时间戳）
```

### 定时发送流程
```
勾选 "定时发送" CheckBox
  → QTimer 按设定间隔触发
  → 自动调用 on_sendBtn_clicked()
```
